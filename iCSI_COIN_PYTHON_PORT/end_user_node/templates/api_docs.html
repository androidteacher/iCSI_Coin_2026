Strict Authentication Required

Every API request must include username and password in the JSON body. Consequently, most endpoints require POST even if retrieving data.
(Alternatively, a valid session cookie is accepted if available.)

API Documentation

Complete reference for interacting with the iCSI Coin node programmatically. Base URL: http://localhost:<PORT>

üîê Authentication

Login (Cookie Method)

POST
/api/auth/login

Authenticates a session and returns a cookie (Optional alternative to explicit body auth).

# Login and save cookie
curl -c cookies.txt -X POST http://localhost:8080/api/auth/login \
     -H "Content-Type: application/json" \
     -d '{"username": "admin", "password": "securepassword"}'

üí∞ Wallet

List Wallets

AUTH POST
/api/wallet/list

Returns all wallets and balances.

curl -X POST http://localhost:8080/api/wallet/list \
     -H "Content-Type: application/json" \
     -d '{"username": "admin", "password": "securepassword"}'

Create Wallet

AUTH POST
/api/wallet/create 
curl -X POST http://localhost:8080/api/wallet/create \
     -H "Content-Type: application/json" \
     -d '{
           "username": "admin", "password": "securepassword",
           "name": "Savings"
         }'

Send Coins

AUTH POST
/api/wallet/send 
curl -X POST http://localhost:8080/api/wallet/send \
     -H "Content-Type: application/json" \
     -d '{
           "username": "admin", "password": "securepassword",
           "from": "1SourceAddr...",
           "to": "1DestAddr...", 
           "amount": 50
         }'
