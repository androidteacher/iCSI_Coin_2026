Understanding Forks

Why do I see multiple blocks at the same height? Is my blockchain broken?

01. The Phenomenon

When you run the Check for Forks query, you might see results like this:

[ { "height": 5492, "count": 2 }, { "height": 5491, "count": 2 }, { "height": 5490, "count": 2 } ]

This count: 2 means that your node has received two valid blocks for the exact same height from the network.

02. Why Does This Happen?

The Race Condition

In a Proof-of-Work system, miners race to solve a puzzle. Sometimes, two miners (Miner A and Miner B) solve it at the exact same time.

Propagation Delay

Your node hears about Block A. Seconds later, it hears about Block B. Both are valid. Your node stores both, creating a temporary "fork" in the road.

03. The Resolution

The blockchain cannot stay split forever. The network follows the Longest Chain Rule.

  • Eventually, a miner finds a new block at the next height on top of one of the candidates (say, Block A).
  • The "A-Chain" becomes longer (has more cumulative work).
  • The entire network switches to Chain A.
  • Block B becomes an Orphan (Stale Block). It remains in your database, but it is no longer "active".
✅

This is Normal Behavior

Seeing forks in your database means your node is healthy and correctly listening to the entire network. It shows the decentralized consensus mechanism at work.
